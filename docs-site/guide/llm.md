# AI 模型设置

AI 模型是数字小祥中负责“理解和思考”的部分。它会阅读你的消息和角色设定，生成符合角色设定的回复，并在合适的时候决定是否调用我们提供的工具。

::: tip 先分清两种“模型”
- **AI 模型（大模型）**：负责理解消息、扮演角色并生成文本回复，例如 DeepSeek、Gemini、Claude 等。
- **声音模型**：负责把文本合成为角色的声音。

这篇文档介绍的是如何设置 AI 模型。如果你想修改角色的声音，请参阅“角色系统 → 角色语音合成”。
:::

## AI 模型会影响什么？

切换不同的 AI 模型，会直接影响你和角色的聊天体验：

- **角色扮演**：能否准确理解角色设定、维持说话风格并自然地回答。
- **回复速度**：不同模型的回答速度可能有明显差异。
- **工具使用**：是否知道何时应该查询时间、天气等工具。
- **稳定性**：能否按照程序的格式要求返回可解析的内容。
- **上下文长度**：能接收并记住多长的聊天记录和角色设定。
- **使用费用**：如果你使用自己的 API，服务商会按 Token 数量计费。不同模型的价格各不相同。

## 支持的接入方式

数字小祥支持接入很多厂家的 AI 模型服务。在设置界面中，你可以看到以下四种选项：

### 我们提供的免费线路

这是默认且最简单的接入方式，**不需要进行任何配置**，打开软件即可直接聊天。

它适合：

- 刚开始使用数字小祥的你；
- 暂时不想去注册其他模型服务账号，想先体验对话功能。

::: warning 使用限制
- 只能使用我们指定的模型（例如 DeepSeek V4 Flash）。
- 共享服务不保证随时都有充足额度。如果你需要长期稳定地使用、想切换其他模型，或自行管理使用费用，建议配置你自己的 API。
:::

### 内置的模型服务商

设置界面中内置了多个常见的模型服务商，例如：

- DeepSeek
- 火山引擎（豆包）
- Moonshot（月之暗面）
- OpenAI
- Anthropic
- Gemini

如果你选择这些服务商，需要先去他们对应的官网注册账号，获取 **API Key**。然后在小祥的设置中填写正确的 **模型名称** 和 **API Key**。部分不常用的服务商位于选择框的“更多”菜单中。

::: tip 没有我想要的模型，怎么办？
如果**模型名称**的下拉列表中没有最新推出的模型，你依然可以直接在框内**手动输入**模型名称。

- 一定要去官网确认<font color=orange>**模型名称的准确写法**</font>。通常模型名称中没有空格，比如 "DeepSeek V4 Pro" 是错的，代码中的准确标识可能是 `deepseek-v4-pro`。
:::

### ModelScope（魔搭社区）

如果你想使用自己的 API 又不想花钱，可以试试魔搭社区。他们每天会提供一定次数的免费 deepseek 模型调用额度。

使用魔搭社区时，你需要填写：

- 网页上显示的完整模型标识，例如 `deepseek-ai/DeepSeek-V4-Pro`；
- 你自己的 API Key。

同样，请一定要看准<font color=orange>**网页上的模型名称到底怎么写**</font>，不要自己猜测。

::: info 免费次数提示
魔搭社区每天的免费额度是针对单个模型的，也就是说你可以使用多种不同的模型，每个模型都有自己的免费次数。
由于程序在处理一次对话时可能会在后台多次调用模型，因此几十次的额度可能很快就会用完。
:::

### 自定义 API（兼容 OpenAI）

如果你使用的服务商不在预设列表中，但他们提供了兼容 OpenAI 格式的接口，你可以选择“自定义 API”。如果你在使用**模型中转站**，通常也应该选择这个选项。

选择此项后，你需要填写：

- **API 地址**：服务商提供的接口地址，通常以 `/v1` 结尾；
- **模型名称**：服务商要求使用的模型标识；
- **API Key**：你的访问密钥。

::: warning DeepSeek 模型配置注意
如果你只是想配置普通的 DeepSeek 模型，请<font color=orange>不要</font>选择此项，直接在**内置的模型服务商**下拉列表中选择“DeepSeek”即可。
:::

## 配置项说明 {#configuration}

### 服务商

服务商是实际提供模型 API 服务的平台。只有选对服务商，程序才能使用对应的请求格式连接模型。请注意，API Key、模型名称和服务商必须相互匹配。例如，DeepSeek 的 API Key 不能直接用于 Gemini。

### 模型名称

模型名称是服务商用于识别模型的准确标识，它不一定等于网页上展示的宣传名称。例如，DeepSeek V4 Pro 在官网上的模型名称是 `deepseek-v4-pro`。
不同服务商对相同模型的名称定义也可能不同，比如 ModelScope 上可能是 `deepseek-ai/DeepSeek-V4-Pro`。请以你使用的平台官方文档为准。

::: warning 不要选择非文本模型
模型列表中可能会出现绘图、语音识别等模型（比如带有 `image` 等字眼的名称）。数字小祥需要的是支持文本聊天的模型，如果你选择了图像生成等非聊天模型，程序将无法正常回复。
:::

### API Key

API Key 是服务商用于识别账号和计算额度的访问密钥。

- **千万不要**把 API Key 发到群聊、公开 Issue 或截图中，也不要把包含 API Key 的配置文件发给别人。
- 如果 API Key 已经泄露，应立即前往服务商后台将其撤销，并生成新的 Key。
- 使用自己的 API 会产生费用，具体计费规则请参考对应服务商的说明。

### API 地址

API 地址（有时也称 Base URL）是程序向模型发送请求的服务器地址。

当你选择 DeepSeek、OpenAI、Gemini 等内置服务商时，程序已经帮你填好了地址，通常不需要手动修改。只有在使用“自定义 API”时，才需要根据服务商的文档来填写。
请不要把普通的网页地址（如 `chat.deepseek.com`）或控制台地址误填为 API 地址。

## 如何选择模型

挑选模型时，除了比较模型是否“聪明”，还需要考虑它与程序的兼容性和稳定性。因为数字小祥会要求模型进行角色扮演、输出固定格式并调用工具。

建议优先选择具备以下能力的模型：

| 模型能力 | 对程序的影响 |
| --- | --- |
| 文本聊天 | 生成角色回复的基本要求。 |
| 指令遵循 | 更稳定地遵守角色设定、语言和回复格式。 |
| JSON 结构化输出 | 降低回复无法解析、从而导致无法合成语音的概率。 |
| Tool Calling | 能否正确使用时间、天气、提醒等程序内置工具。 |
| 上下文长度 | 较长的窗口可以在长对话中保留更多聊天记录和角色信息。 |
| 推理模式 | 可在复杂问题中投入更多算力，但回复通常更慢、更贵。 |

### 按使用需求选择

**日常角色聊天**

优先考虑角色扮演自然、回复速度快、价格适中的模型。日常闲聊通常不需要持续开启高强度的推理模式。

**经常使用工具**

建议选择明确支持 Tool Calling（工具调用）的模型。如果模型不能正确生成调用请求，它仍然能聊天，但可能无法可靠地使用查询或换装等功能。

**长时间连续聊天**

建议选择上下文窗口较大的模型。程序会向模型提供角色设定、部分历史聊天记录以及工具的返回结果，这些都会占用上下文。

**使用本地模型**

可以通过 Ollama、vLLM 等软件以兼容 OpenAI API 的方式接入本地模型。请注意，本地运行不代表完全离线：如果模型使用了网络搜索、天气等工具，依然会访问网络。
本地模型的表现受限于你的电脑性能，较小的模型更容易出现忘记角色设定、工具调用失败或格式错误等问题。

## 工具调用

数字小祥不仅会让模型生成文字，还会向模型提供一组可以调用的工具。目前可用的工具包括：

- 获取当前日期和时间；
- 查询 CPU、内存和 NVIDIA GPU 状态；
- 搜索网络信息；
- 查询天气；
- 创建定时提醒；
- 打开随机抽签界面；
- 查看和切换部分角色的 Live2D 服装；
- 浏览、读取和搜索你指定的本地文件。

模型会根据你们的对话内容，自行判断是否需要调用工具。有时它可能会在一次回复中进行多轮工具调用，所以当你觉得等待时间变长时，请耐心等待。

::: tip 工具能力取决于模型
程序虽然提供了工具，但不代表所有模型都能正确使用。模型需要支持 Tool Calling 并能理解工具参数。指令遵循能力较弱的模型可能会忽略工具或填错参数。
部分工具也受限于你的电脑环境。比如，电脑没有 NVIDIA 显卡时，就无法获取相应状态；断网时，查询天气等工具也会失败。
:::

## 回复格式与“显示无法解析的模型原文”

为了让程序能够拆分文本、判断情绪、选择动作并生成语音，模型的最终回复必须符合程序要求的结构化格式（通常是 JSON）。

如果模型没有按格式输出，程序会尝试让它重新整理；如果仍然无法解析，这次回复可能会失败。

设置中的 **“显示无法解析的模型原文”** 选项允许在解析失败时，直接显示模型返回的原始文本。我们建议这仅作为临时排查问题的手段：

- 原始文本可能缺少情绪和翻译信息，导致语音合成效果较差。
- 这个选项只是绕过了解析失败的限制，并没有真正解决格式兼容的问题。

如果某个模型经常依赖这个选项才能显示回复，建议你更换一个指令遵循和结构化输出能力更好的模型。

## 推理模式

在主聊天界面的输入框中，你可以修改当前的推理设置。每次对话的推理设置都是独立的。

### 开启推理

- **自动**：由模型或服务商自动决定是否开启推理。
- **思考**：请求模型启用推理模式。
- **快速**：请求模型关闭推理模式。

### 推理强度

可选的强度包括：默认、极低、低、中、高、极高。
强度越高，模型越倾向于投入更多算力处理复杂问题，但也可能会增加等待时间和 API 费用。

::: info 模型可能忽略设置
并非所有模型或服务商都支持推理开关与推理强度。对于不支持的模型，程序会自动忽略这些设置，以保证对话能继续进行。
例如，DeepSeek V4 系列模型只支持“高”和“极高”的强度；如果你设置得低于“高”，程序会自动按“高”强度处理。
:::

一般建议：

- 日常闲聊使用“自动”或“快速”；
- 面对复杂分析、多步骤任务或工具调用不稳定时，可以尝试“思考”；
- 没有明确需求时，强度保持“默认”即可。

## 温度与核采样

在启动配置程序中，还提供了“模型温度”和“模型核采样”两个高级参数。

### 模型温度（Temperature）

温度控制回复的随机程度：

- 较低：回复通常更稳定、保守和可预测；
- 较高：回复更活泼、发散，但也更容易跑题或破坏输出格式。

### 模型核采样（Top-p）

Top-p 控制模型生成内容时考虑的候选词范围：

- 较低：输出范围更集中；
- 较高：允许更多样的表达。

::: warning 建议保持默认值
普通用户通常不需要修改这两个参数，保持默认即可。如果需要调整，建议一次只修改其中一个，并进行微调。部分模型可能不支持这些参数，程序会尽量忽略它们。
:::

## 上下文与长对话

模型在每次回复时，都会收到所有的聊天记录，其中包含角色设定、聊天历史、运行设置以及必要的工具结果。

这些内容共同占用了模型的上下文窗口。对话越长，占用越多。主界面会根据模型信息估算当前的上下文使用情况。

如果模型支持的上下文较小，可能会出现以下问题：

- 模型忘记了你们之前提到的信息；
- 服务商返回“上下文过长”或类似的错误导致回复失败。

选择支持更长上下文的模型可以缓解这些问题，但也可能增加费用和等待时间。

## 设置何时生效

在启动配置程序中保存 AI 模型设置后，主程序会自动重新读取配置，你不需要重启软件。
推理模式和推理强度属于当前对话的独立设置，会在聊天中分别保存。在生成回复的过程中修改设置，只会影响下一次发送的消息。

## 隐私与数据发送

使用在线的 AI 模型服务时，程序不可避免地需要将生成回复所需的信息发送给所选服务商，这可能包括：

- 你的当前消息；
- 为了保持连贯而提供的历史聊天记录；
- 当前角色设定；
- 语言和格式要求；
- 工具调用的结果。

因此，**请千万不要在对话中发送密码、身份证等敏感的个人信息**。不同服务商的隐私政策不同，请以对应平台的说明为准。

使用本地模型可以减少聊天内容被发送给外部服务器的情况，但网络搜索、天气等在线工具在执行时仍可能向相应服务发送查询内容。

## 还需要其他帮助吗？

如果你在设置时遇到了问题，可以参考以下页面：

- **需要填写自己的模型服务**：参阅“常用操作 → 配置自己的大模型 API”。
- **出现认证失败、连接失败、请求超时或余额不足**：参阅“故障排除 → API 调用失败”。
- **角色没有返回任何内容**：参阅“故障排除 → 角色没有回复”。
- **回复正常但没有声音**：这通常不是 AI 模型的问题，请参阅“故障排除 → 角色没有声音”。
