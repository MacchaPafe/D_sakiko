# 问题反馈

如果你在使用数字小祥时遇到了问题，可以尝试以下步骤：

1. 先在 [常见问题](faq.md) 页面中看看有没有对应的解答；
2. 如果问题仍未解决，随时可以向我们反馈并寻求帮助。

在向我们反馈之前，请参考以下指南准备相关信息。

## 需要提交的信息

为了帮助我们快速定位并修复问题，在反馈时请务必附上程序的日志文件（Log）。你需要准备以下两个文件：

- `logs/app.log`：程序的主日志，记录了软件运行的整体状态和提示信息。
- `logs/error.log`：错误日志，专门记录程序运行过程中发生的异常或崩溃详情。

::: tip 寻找历史日志
如果你遇到的问题并不是今天发生的，日志文件的名称会带有当天的日期后缀。比如，如果问题发生在 2026 年 5 月 31 日，你需要寻找 `logs/app.log.2026-05-31` 和 `logs/error.log.2026-05-31`。
:::

::: tip 更新相关的问题
如果你的问题是在更新软件时出现的，请额外提供更新日志：
- 对于 3.1.0 及之后的版本，请寻找 `logs/update/` 文件夹下的文件；
- 对于更早的版本，请寻找 `.updates/logs/` 文件夹下的文件。
:::

## 反馈渠道

你可以选择以下任意一种方式与我们联系。**无论你选择哪种方式，都请记得附上前面提到的日志文件**，这会对我们排查问题有极大的帮助。

1. **GitHub Issue**：你可以访问 [项目 Issue 页面](https://github.com/MacchaPafe/D_sakiko/issues)，点击 "New issue" 按钮。请在里面详细描述你遇到的问题，并将日志文件附带上传。
2. **QQ 群交流**：如果你已经加入了项目的交流群（QQ 群号：1026822753），可以直接在群里提问，或者联系群主及管理员进行反馈。
