# macOS 安装

由于 macOS 系统的特殊性，在第一次使用前，你需要运行一个安装程序。不过别担心，这个安装程序是全自动的，它会帮你搞定 Homebrew、Python、uv 以及各种依赖，不需要你自己去敲复杂的代码。

::: warning 我们的支持范围
目前的软件包仅正式支持 **Apple M 系列芯片** 且系统在 **macOS 14 Sonoma 或更高版本**的 Mac。如果你用的是 Intel 芯片的 Mac，或者系统版本低于 macOS 14，很可能装不上。
:::

- **百度网盘**：https://pan.baidu.com/s/1UrOc8sdQd4DeYtjo0ISTuA?pwd=m49g 提取码: m49g
- **天翼云盘**：https://cloud.189.cn/t/Q7Z7r2nYNvyu （访问码：yf5b）
- **Google Drive**：https://drive.google.com/drive/folders/1jWgKRWXvpRIgyiYpkVgk7vXGMFL-kiyz?usp=sharing

解压的时候，推荐使用压缩软件 [Keka](https://www.keka.io/zh-cn/)。（提醒：它在 Mac App Store 里是收费的，但你去它的官网就能下到一模一样的免费版）。

解压后，请把整个软件文件夹搬到一个你打算长期存放的地方。千万别在安装完之后手滑删掉里面的 `.venv`、`GPT_SoVITS` 等文件夹。

在接下来的安装过程里，程序需要联网下载大概 1 到 2 GB 的东西。安装全搞定后，整个软件大约会占你 8 到 10 GB 的空间。所以动手前，请保证你的网络通畅，并且硬盘空间足够。

## 2. 运行安装程序

双击软件文件夹里的：

```text
安装程序.command
```

双击后，会弹出一个终端窗口，它就开始自动检查你的电脑环境了。

### 提示“无法打开”怎么办？

这是因为 macOS 对没有开发者签名的脚本有比较严格的安全限制。只要你确认软件是从我们官方给的链接下载的，就可以按下面的步骤放行：

1. 打开电脑的“系统设置”；
2. 找到并进入“隐私与安全性”；
3. 往下滚，找到关于拦截 `安装程序.command` 的提示；
4. 点击“仍要打开”；
5. 按照弹窗提示，输入你电脑的开机密码确认一下。

放行过一次后，以后再点它就不会被拦了。

## 3. 选择下载服务器

终端弹出来后，第一件事会问你是不是在中国大陆：

- 如果你在国内：在键盘上敲个 `y` 然后按回车；
- 如果你在海外：敲个 `n` 按回车。

这只是为了帮你配置一个下载加速镜像（让下载速度快点），没有别的作用。

## 4. 坐和放宽，等待自动安装

接下来你就不用管了，安装程序会自动帮你完成：

- 检查并安装 Homebrew；
- 安装 Python 3.11；
- 安装 uv 包管理器；
- 创建一个不会弄脏你电脑的独立 `.venv` 运行环境；
- 把项目需要的各种包都装好；
- 搞定部分烦人的 macOS 权限问题。

这个过程快的话几分钟，慢的话可能要半小时以上，全看你的网速和电脑性能。**在这期间不要关掉那个终端窗口，也别让电脑休眠了**。

### 突然系统弹窗要安装命令行工具？

如果你的电脑比较新，缺了一些基础的编译组件，安装程序可能会触发系统的弹窗，让你装 Apple 的 Xcode 命令行工具。

这时候你只需要：
1. 在弹窗上点“安装”；
2. 耐心等它下载并装完；
3. 回到刚才那个终端窗口，按它的提示继续；
4. 如果终端里的安装程序已经提前结束或者退出了，就再双击运行一次 `安装程序.command`。

### 怎么知道我装成功了没？

当终端跑到最后一行，对你说：

```text
安装完成！
```
那就大功告成了！
如果你一直没看到这句话，别犹豫，再去双击运行一次安装程序。它有自动检查功能，只会补齐没装好的部分，不用你去删掉重装。

## 5. 启动主程序

安装成功后，以后你想找角色聊天，就双击：

```text
运行主程序.command
```

第一次运行这个脚本的时候，苹果可能又会跳出来说“身份不明的开发者”。老规矩，去“隐私与安全性”里点一下“仍要打开”就行。

启动后，那个终端窗口会一直留着，**软件运行期间别关它**。等聊天窗口和 Live2D 纸片人弹出来后，你就可以去看 [第一次启动](./first-run.md) 的教程了。

## 6. 在桌面上做个快捷入口

如果你觉得每次都要进文件夹找启动脚本太麻烦，可以给它做个“替身”（类似 Windows 的快捷方式）：

1. 右键点击 `运行主程序.command`；
2. 选择“制作替身”；
3. 把那个长得一样的替身文件拖到桌面上。

记住，真正的软件文件夹还是要留着的，替身只是个传送门。

## 常见的安装小麻烦

### 终端好久都没动静了，是不是卡死了？

有时候 Homebrew 和依赖包在下载时就像没动静一样，只要终端里偶尔还在跳字，或者进度条还在动，就耐心等。如果真的是半小时都不动，可能是网断了，你再重新运行一次安装程序就好。

### 启动时报错说找不到虚拟环境

这说明前面的环境 `.venv` 没建成功，或者是你不小心把它删了。解决办法很简单：再双击运行一次 `安装程序.command`。

### 软件闪退了

如果它突然崩了，请把你看到终端里的错误文本全部复制下来（别只截图最后一行）。如果来找我们反馈问题，别忘了带上日志文件。

### 我能把它固定在程序坞（Dock）上吗？

目前不行。因为现在的启动入口是个 `.command` 脚本，不是那种正式的 `.app` 应用，所以苹果不允许你把它钉在下面。你可以用上面说的那种“桌面替身”的方法。
