# 更新与维护

数字小祥支持在程序内检查、下载并自动安装更新。一般情况下，你不需要自己去下载新的安装包，也不需要在每次更新前繁琐地备份数据。

::: tip 自动更新不会破坏你的数据
更新补丁不会修改你的配置文件、聊天记录或自定义角色。不过，如果你打算重新安装、换电脑，或者已经存了很多自定义角色和模型，我们仍然强烈建议你阅读 [备份、迁移与重新安装](#备份-迁移与重新安装)，备份一下自己的重要数据。
:::

## 更新程序

### 更新前的准备

动手更新前，请先确认：

- 角色当前没有正在说话、生成声音或下载东西；
- 你的网络连接是稳定的；
- 软件所在的硬盘还有足够的剩余空间；

更新程序在安装新版本前，会自动保存当前的聊天记录并关闭主程序。为了稳妥起见，你也可以先手动点一下聊天窗口上方的“保存聊天记录”按钮。

### 自动检查更新

每次你打开数字小祥时，程序都会自动去检查有没有新版本。

- 如果发现了新版本，聊天窗口的顶部会跳出提示。点击“更多功能“-”检查更新“就能看到具体更新了什么内容。
- 如果没有新版本，或者网络不好连不上更新服务器，它会在后台安静退出，绝对不会打断你的正常聊天。

请放心，自动检查仅仅是“看看有没有新版本”，在你点头确认之前，它绝不会擅自下载或安装任何东西。

### 手动检查更新

如果你想自己手动查一查：

1. 打开主聊天窗口；
2. 点击“更多功能”；
3. 点击“检查更新”；
4. 等待几秒钟，让程序连接更新服务器。

如果当前已经是最新版本了，屏幕上会提示“当前已是最新版本”。

### 下载并安装

发现新版本后，弹出的更新窗口会告诉你新版本的版本号、更新包有多大，以及具体更新了什么功能。

1. 看一眼更新说明，决定是不是现在就要升；
2. 点击“下载并安装”；
3. 在弹出的确认窗口里点“是”；
4. 等待更新包下载完成；
5. 下载完之后，主程序会自动关闭，并在后台开始安装；
6. 安装成功后，数字小祥会自动重新启动。

::: warning 安装开始后千万不要强制结束
主程序关闭并进入安装阶段后，请绝对不要关机、强制关掉那个黑色的终端窗口，或者去删软件文件夹。安装通常需要几分钟，在 macOS 上更新环境依赖时可能会花更长的时间。
:::

在更新过程中，程序会仔细比对更新包和当前版本是否匹配。如果发现不对劲（校验失败），它会自动放弃，不会把你的软件搞坏。

### 怎么确认更新成功了？

更新完成后，程序通常会自动重新打开。你可以再次点进“更多功能 → 检查更新”：

- 如果提示“当前已是最新版本”，恭喜你，更新成功了！
- 如果还在提示有同一个新版本，或者程序死活没有重新启动，请看下面的 [更新失败怎么办](#更新失败怎么办)。

### 哪些情况没法用自动更新？

如果遇到以下情况，自动更新可能会罢工：

- 你曾经手动改过程序内部的代码文件，导致它和更新包要求的前置条件对不上；
- 你用的版本实在太老了，已经低于自动更新支持的底线；
- 你的网络暂时连不上我们的更新服务器。

## 更新失败怎么办

别慌，更新失败不一定意味着软件已经坏了。下载和安装是两个不同的阶段，处理方法也不一样。

### 检查或者下载失败

如果报错发生在主程序关闭之前，那说明程序文件根本还没动过。常见的现象有：

- 检查更新时报错；
- 下载进度一直卡在 0；
- 提示网络连接超时；
- 你自己点了取消，停止了下载。

你可以按顺序试试：

1. 确认一下你的浏览器能不能正常上网；
2. 如果你开了加速器或代理软件，先关掉试试，或者换个 Wi-Fi；
3. 关掉更新窗口，重新点一次“检查更新”；
4. 把数字小祥彻底关掉，重新打开再试。

### 程序关闭后，再也没有重新启动

下载完之后，数字小祥会关掉主程序，让后台的一个独立小程序去替换文件。只有替换成功了，它才会把主程序再叫醒。

如果程序关掉后一直没反应：

1. 先多等几分钟（尤其是用 Mac 的朋友）；
2. 不要疯狂连续双击启动图标；
3. 看一眼还有没有正在跑着代码的黑色终端窗口；
4. 如果啥也没了，就按你平时的方式再启动一次：
   - Windows：双击 `run.bat`
   - macOS：双击 `运行主程序.command`

如果你发现软件还能正常打开，而且还是旧版本，那说明安装失败时它已经成功“自动抢救（回滚）”了。你可以再试着更新一次。

### 更新后软件彻底打不开了

#### Windows 用户看这里

先检查这三点：

- 软件的路径里是不是混进去了中文或特殊字符？
- 软件文件夹是不是被锁了（没有写入权限）？
- 你电脑的硬盘是不是塞满了？

如果杀毒软件把某些文件当成病毒隔离了，请去把它放出来，然后再试。如果整个文件夹已经被搞得乱七八糟了，建议你直接使用 [保留数据重新安装](#保留数据重新安装) 大法。

#### macOS 用户看这里

Mac 在更新时，如果底层的环境依赖变了，即使程序文件自动抢救回来了，虚拟环境（`.venv`）也可能已经被破坏了。

这时候请在软件文件夹里，双击运行一下这个：

```text
安装程序.command
```

这个安装脚本会重新去检查 `.venv` 并且补齐缺少的包。它**不会**删掉你的聊天记录和角色数据。跑完之后，再双击 `运行主程序.command` 试试。

### 自动抢救（回滚）的极限

普通的安装错误都能触发自动抢救，但遇到下面这些极端情况，它可能就无能为力了：

- 更新到一半，电脑突然断电或者你强行关机了；
- 硬盘坏了，或者空间彻底满了；
- 文件夹没有写入权限；

遇到这些倒霉事，千万别自己乱拷文件去覆盖。保留好现在的软件文件夹，老老实实按照 [备份、迁移与重新安装](#备份-迁移与重新安装) 的流程下一个全新的版本，才是最稳妥的。

### 去哪找更新日志（Log）？

更新日志存放在软件文件夹下的：

```text
logs/update/
```

文件名里一般都写着更新前后的版本号和具体时间，比如：

```text
update_3.0.3_to_3.1.0_20260614_120000.log
```

如果是比较老的版本（3.1.0 以前），日志可能藏在这里：

```text
.updates/logs/
```

::: warning 提交日志须知
如果你要找我们求助，请把**整个**日志文件直接发给我们，千万不要只发一小段截图，或者只复制最后几行文字。
:::

请把出问题的那个软件文件夹原封不动地留着，准备好下面这些东西：

- `logs/update/` 里最新的一份更新日志；
- `logs/app.log`；
- `logs/error.log`；
- 你用的什么系统，更新前是哪个版本，想升哪个版本；
- 更新前后你都干了什么，以及你已经试过了哪些抢救方法。

带着这些信息，按照 [问题反馈](./feedback.md) 里的方法联系我们，我们会尽力帮你搞定。

## 备份、迁移与重新安装

因为平时的自动更新不会乱动你的数据，你不需要每次更新时都备份一次。

但在下面这几种情况里，**强烈建议你做好备份**：

- 你打算彻底重新安装，或者想手动下载完整的安装包来升级；
- 你换了新电脑，或者想把软件挪到另一个硬盘里；
- 你辛辛苦苦给角色换了新衣服、新背景，或者加了一大堆自定义的 live2d 服装，想备份一下自己添加的东西；

### 动手前，先退出程序

复制文件之前，请务必：

1. 在聊天窗口点一下“保存聊天记录”；
2. 点击“更多功能 → 退出程序”，等待程序退出；
3. 等到聊天窗口、Live2D 窗口和黑色的终端窗口全部消失。

如果你趁程序还在跑的时候去强行复制文件，很可能复制到一份写了一半的残次品。

### 我的数据都存在哪了？

数字小祥的所有数据都存在了自己的软件文件夹中，不会去偷偷塞进你电脑的“文档”或者 C 盘深处。

| 要备份什么 | 你需要复制的文件或文件夹是哪个 |
| --- | --- |
| 各种设置、API Key、你的用户人设 | `d_sakiko_config.json` |
| 聊天文字记录 | `reference_audio/all_conversation.json` |
| 你设定的头像图片 | `avatar/`（可能不存在） |
| 角色设定、Live2D 模型、衣服和背景 | `live2d_related/` |
| 声音模型、参考音频、以及生成的语音 | `reference_audio/` |
| 你自己导入的字体 | `font/` |

::: danger 配置文件里有你的 API Key！
`d_sakiko_config.json` 里面存着你花钱买的大模型 API Key。千万不要把它传到网盘、发到群里或者截屏给别人看。就算找我们反馈问题，也别把这个文件发给我们！
:::

### 我到底该备份哪些东西？

#### 方案 A：只要聊天记录和设置（最基础）

你至少需要把这两个拷走：

```text
d_sakiko_config.json
reference_audio/all_conversation.json
```

这样能保住你的大部分设置、你的人设以及你们聊过的所有文字。

::: warning 声音可能听不到了
之前的聊天语音是存在 `reference_audio/generated_audios_temp/` 里的。如果你只备份了 `all_conversation.json`，文字虽然还在，但你回去点以前的聊天记录时，角色就不会发声了。
:::

#### 方案 B：保住我自己添加的角色（进阶）

如果你自己加过角色或者改过衣服，还必须把这两个文件夹带走：

```text
live2d_related/
reference_audio/
```

注意，这两个文件夹是**捆绑销售**的：

- `live2d_related/<角色名字>/` 里装着角色长什么样（衣服、Live2D、文字设定）；
- `reference_audio/<角色名字>/` 里装着角色怎么说话（声音模型和参考音频）。

只拷走一个的话，角色就会变成只张嘴不出声的哑巴，或者只听见声音却看不见人。

#### 方案 C：备份全部数据

只需要把上面表格中提到的文件夹都拷贝到其他地方，就可以备份你的所有数据了。总体来看，你需要拷贝这些内容：

- `d_sakiko_config.json`：你设置的内容；
- 整个 `reference_audio` 文件夹：所有角色的声音模型和你们的聊天记录；
- 整个 `live2d_related` 文件夹：所有角色的 Live2D 模型、衣服和设定； 
- `avatar/` 文件夹：你设置的头像图片。如果这个文件夹不存在，没关系，可以不管它。
- `font/` 文件夹：你导入的字体（如果有的话）。

### 我们推荐的备份姿势

1. 在别的地方（比如 D 盘、移动硬盘里）建个新文件夹；
2. 把需要的文件复制进去；
3. 给这个文件夹改个好记的名字，最好带上日期和版本号，比如：
   `D_sakiko_backup_3.1.0_2026-06-14`
4. 点进去看一眼，确认那些关键的 `.json` 文件和角色文件夹是不是都在；
5. 如果想长期保存，还可以把它打包成一个压缩包。

### 保留数据重新安装

所谓的“重新安装”，**千万不要**把最新下载的压缩包直接解压覆盖到旧文件夹里。正确的做法是：保留旧的文件夹，在别的地方装个全新的，然后再把你的“家当”搬过去。

如果遇到这些情况，我们推荐你重新安装一次：

- 自动更新一直没法用；
- 程序文件被你不小心搞坏了；
- 杀毒软件把环境弄得乱七八糟；
- 在 macOS 上反复修复环境还是启动不了；
- 软件总出各种奇葩 Bug，想找个干净的新版本来排查。

#### 第一步：保留并改名旧文件夹

先把出问题的数字小祥正常退出。然后把那个软件文件夹改个名字，让你一眼就知道它是旧的，比如：

```text
D_sakiko_old_3.0.3
```

**千万先别删它！** 在新版本没跑通之前，这个旧文件夹就是你最后的救命稻草。

#### 第二步：搞一个全新的版本

- **Windows**：把刚下载好的最新版安装包，解压到一个**全新**的文件夹里。绝对不能解压到刚才那个改过名的旧文件夹上。记住路径里不能有中文。
- **macOS**：把安装包解压到新文件夹，双击运行里面的 `安装程序.command` 等它跑完。

#### 第三步：蚂蚁搬家（迁移数据）

确认新程序没在运行，然后开始把旧文件夹里的东西往新文件夹里搬。

建议你按照这个顺序复制并覆盖过去：

1. 你的自定义角色、衣服、背景等资源；
2. `avatar/` 头像文件夹；
3. `reference_audio/all_conversation.json` 聊天记录；
4. 最后再把 `d_sakiko_config.json` 复制过去。

先让文件就位，再复制设置文件，这样里面的路径才不会指错路。

::: details 如果我自定义了一堆东西，具体该搬哪些？（点击展开）
- **你完全自己捏的新角色**：复制 `live2d_related/<角色文件夹>/` 和对应的 `reference_audio/<角色文件夹>/`。
- **给官方角色买的新衣服**：把那个角色目录里的 `extra_model/` 文件夹拷过去。
- **改过角色的名字或人设**：把对应目录下的 `name.txt` 和 `character_description.txt` 拷过去。
- **自己训练的或者换过的声音模型**：拷对应的 `reference_audio/<角色文件夹>/`。
- **自己找的背景图片**：拷你塞进 `live2d_related/` 里的那些图片文件。
- **自己找的好看字体**：拷整个 `font/` 文件夹。

**千万注意：** 绝对不要把 `GPT_SoVITS/`、`runtime/` 或者 `tools/` 这些文件夹复制回去！这些是程序的执行代码，不是你的数据，如果你把旧的代码塞进新版本里，新版本马上就会“退化”回有 Bug 的旧版本。
:::

#### 第四步：启动验收

双击新版本启动，挨个确认一遍：

- 设置里的 AI 模型选对了没？API Key 还在不在？
- 聊天记录是不是都回来了？
- 头像和你的身份设定对不对？
- 角色列表里有没有你的自定义角色？
- 角色的衣服和背景还能不能正常显示？
- 角色还能不能开口说话？

如果你发现背景图、背景音乐或者衣服不见了，可能是因为它们还记着旧文件夹的绝对路径。你去设置里重新选一次就行了。

#### 第五步：送旧迎新

在新版本里舒舒服服地聊了几天，确认什么都没少之后，你就可以放心地把刚才那个改过名的旧文件夹彻底删掉了。

### macOS 的轻量级急救（只修复运行环境）

在 Mac 上，如果你确认程序文件和你的聊天数据都没动过，软件只是报错说找不到 `.venv` 虚拟环境、少了哪个 Python 包、或者更新依赖失败了，那大可不必大动干戈去完整重装。

你只需要在原来的软件文件夹里，重新双击跑一次这个：

```text
安装程序.command
```

它是个聪明的修理工，会自动帮你把 `.venv` 环境重新建好、补齐缺少的依赖，并且**绝对不会**去碰你的 `d_sakiko_config.json` 或是那些存着聊天记录的文件夹。

### 换电脑时的温馨提醒

- **Mac 用户注意**：千万别把旧电脑上的 `.venv/` 文件夹直接拷给新电脑用，大概率是用不了的。
- 如果你的设置里引用了电脑上其他地方的图片或音乐（比如在 C 盘里的某张背景图），这些文件是不会跟着软件备份一起走的，你得自己把它们找出来拷到新电脑上。
- 换电脑后，因为硬盘盘符或者用户名变了，原来设置好的绝对路径可能会失效。这时候只要去设置里重新选一次背景或者参考音频就可以了。

## 怎么卸载数字小祥？

数字小祥是个非常干净的“绿色软件”，程序、设置、聊天记录和角色数据全都在那一个文件夹里。

::: warning 删文件夹 = 删光所有数据
一旦你把软件文件夹删了，你和角色的所有回忆也就灰飞烟灭了。如果你想着以后还会再下回来，千万记得先看看上面的 [备份、迁移与重新安装](#备份-迁移与重新安装)，把备份文件存到软件文件夹外面去。
:::

### Windows 用户怎么卸载

1. 在聊天窗口点击“更多功能 → 退出程序”；
2. 确认聊天窗口、Live2D 窗口以及那个黑色的终端窗口都已经关掉了；
3. 直接选中整个数字小祥的软件文件夹，选择删除；
4. 把你在桌面上建的 `run.bat` 快捷方式删掉；
5. 右键清空回收站，搞定。

数字小祥不会进入 Windows 的“已安装的应用”列表里，所以你不用去控制面板里找它。
如果电脑提示“文件正在被使用无法删除”，不要硬删，重启一下电脑再删就行了。

### macOS 用户怎么卸载

1. 在聊天窗口点击“更多功能 → 退出程序”；
2. 确认所有窗口（包括黑色的终端窗口）都关掉了；
3. 把整个数字小祥的软件文件夹拖进“废纸篓”；
4. 把桌面上那个 `运行主程序.command` 的替身也拖进废纸篓；
5. 清空废纸篓，搞定。

::: tip 你的在线账户和余额都在
请放心，把数字小祥删了，不会影响你在 DeepSeek、豆包或者魔搭社区上注册的账号，你花钱充值的 API 余额也不会丢。如果你以后再也不打算用了，记得自己去那些平台的官网把生成的 API Key 删掉就行。
:::
