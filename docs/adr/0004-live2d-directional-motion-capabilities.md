# Live2D 方向动作组只表示真实方向能力

Live2D V3 归一化后的项目标准动作组保留基础 `{group}` 作为默认播放入口，方向变体 `{group}_L`、`{group}_C`、`{group}_R` 只在模型存在显式、非空的对应方向动作时写入；无方向动作不再广播成三份方向组。运行时仍可在请求方向组缺失时回退到基础组，但这种 fallback 不算方向动作支持；小剧场只在左槽模型支持 `L` 且右槽模型支持 `R` 时显示面对面动作朝向，并在当前模型组合不支持时把本次 payload 降级为 `screen` 而不改写用户偏好配置。

基础 `{group}` 按项目标准动作组逐组建立：优先使用无后缀动作；没有无后缀但有 `_C` 动作时使用 `_C` 作为默认；不把 `_L/_R` 混入默认组。动作编辑器只展示归一化结果中真实存在的动作组，不为了编辑便利补齐空方向组；方向能力查询只统计持久 `model3.json`/`model.json` 动作组，不受预览或动态加载 motion 影响。由于旧 `NORMALIZED_MODEL3_VERSION = 1` 语义尚未发布，版本号不升级，本地开发期旧模型由维护者手动清理。

标准动作组 fallback 也按方向隔离：基础组只从其他基础组补，`_L/_C/_R` 只从对应方向变体补；基础组和全局默认池都不使用 `_L/_R` 动作兜底。这样方向变体组一旦存在就代表真实方向能力，普通模式也不会因为缺少默认动作而随机播放侧向动作。
