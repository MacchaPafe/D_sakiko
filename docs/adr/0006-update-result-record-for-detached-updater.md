# 使用更新结果记录报告 detached 更新器失败

自动更新器需要在 macOS `.command` 启动场景下与主程序分离运行，因此主程序不能可靠地等待子进程并实时显示失败。项目决定由更新器在收到 `--status-file` 时自行维护 `logs/update/last_update_result.json`，记录 `running`、`success` 或 `failed`、回滚状态、相对日志路径和是否已提示用户；主程序下次启动时只在 `failed` 且未提示时显示一次面向用户的回滚说明，并提供在文件管理器中显示日志文件的入口。

完整诊断信息仍保存在 `logs/update/*.log` 中，更新结果记录不保存技术错误摘要、traceback 或文件级处理结果。这样避免解析自然语言日志，也避免向普通用户展示低行动价值的内部错误细节，同时保留开发者排查所需的日志文件。
