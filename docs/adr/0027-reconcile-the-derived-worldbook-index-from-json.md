# 从权威 JSON 对账派生世界书索引

用户编辑先原子持久化 JSON，再按稳定条目身份更新索引；失败不回滚 JSON，而将对应包标记为不可检索并等待重试或重建。普通编辑按条目增量同步，官方更新按包对账，索引 Schema、投影或嵌入模型指纹变化时全量重建；程序在主界面可用后后台按需执行，并暴露基础设施和包级 revision 的结构化 readiness，而不把异常伪装成零命中。Qdrant point 使用条目 UUID，携带包身份、条目 revision 和来源层的统一 envelope，再叠加类型模块的扁平投影；这避免跨存储伪事务和陈旧内容泄漏，但要求调用方在依赖包 dirty 或 failed 时暂停相关检索。
